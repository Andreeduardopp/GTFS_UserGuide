### Documentation

The documentation component of this project provides a structured, progressive learning path for public transport professionals transitioning from GTFS to NeTEx. It is designed as a self-contained training curriculum, organized into nine sequential modules that build incrementally upon one another. Each module pairs theoretical explanations with annotated XML examples derived from real operational data—specifically the STCP Line 200 (Bolhão–Castelo do Queijo) in Porto—ensuring that every concept is grounded in a verifiable, real-world context.

The pedagogical strategy is anchored in a consistent comparative approach: for every NeTEx construct introduced, the corresponding GTFS equivalent is presented side-by-side, explicitly highlighting what NeTEx adds in terms of expressiveness, interoperability, and regulatory compliance. This allows practitioners already familiar with GTFS to leverage their existing knowledge while systematically acquiring the more rigorous data modeling capabilities that NeTEx demands.

The modules are organized as follows:

- **Module 0 — Prerequisites and Context:** Establishes the foundational context for the entire curriculum. It defines NeTEx and its relationship to the Transmodel conceptual framework (CEN EN 12896), situates the standard within the broader European ecosystem (Transmodel → NeTEx → SIRI → NAP), and outlines the regulatory obligations imposed by EU Delegated Regulation 2024/490 (MMTIS). A structured comparison between GTFS and NeTEx is provided across multiple dimensions—format, conceptual model, coverage, identifiers, versioning, and extensibility—giving readers a comprehensive overview before engaging with the technical content.

- **Module 1 — Core Concepts:** Introduces the structural foundations of NeTEx: the object-oriented entity model inherited from Transmodel, the `VersionFrame` architecture that organizes data into typed containers (`ResourceFrame`, `ServiceFrame`, `SiteFrame`, `TimetableFrame`, `FareFrame`), and the hierarchical identifier and versioning system that ensures global uniqueness and temporal traceability. The module demonstrates how a NeTEx file is structured from the `PublicationDelivery` envelope down to individual entity references.

- **Module 2 — Transport Network:** Covers the modeling of transport network topology, decomposing it into three distinct layers: the commercial layer (`Line`, `Direction`), the topological layer (`Route`, `RoutePoint`, `ServiceJourneyPattern`), and the physical layer (`StopPlace`, `Quay`). This module explains the separation between logical stop points (`ScheduledStopPoint`) and physical infrastructure—a distinction absent in GTFS—and provides a complete mapping of `routes.txt` and `stops.txt` into their NeTEx equivalents.

- **Module 3 — Timetables and Calendars:** Addresses the modeling of vehicle journeys and service calendars. It introduces the `ServiceJourney` entity with its associated `TimetabledPassingTime` records, and the calendar system comprising `DayType`, `OperatingDay`, and `DayTypeAssignment`. The module demonstrates how `trips.txt`, `stop_times.txt`, and `calendar_dates.txt` from GTFS translate into the NeTEx `TimetableFrame` and `ServiceCalendarFrame`, emphasizing the reusability of journey patterns and the expressiveness of the day-type model for representing complex service variations.

- **Module 4 — Multimodality and Physical Infrastructure:** Explores the NeTEx approach to modeling physical transport infrastructure through the `SiteFrame`. It details the `StopPlace` → `Quay` → `BoardingPosition` hierarchy, the `PassengerStopAssignment` mechanism that bridges logical and physical layers, and the use of `PathLink` elements for modeling pedestrian transfer paths between modes. Using the Bolhão intermodal node in Porto (bus and metro) as a case study, the module demonstrates how NeTEx enables precise multimodal interchange modeling that is structurally impossible in GTFS.

- **Module 5 — Accessibility:** Presents the NeTEx multi-dimensional accessibility model, which evaluates vehicle and infrastructure constraints independently through the `AccessibilityAssessment` and `AccessibilityLimitation` elements. Unlike the binary `wheelchair_accessible` / `wheelchair_boarding` fields in GTFS, NeTEx supports granular assessments across six dimensions (wheelchair access, step-free access, escalator-free access, lift-free access, audible signals, and visual signs), each with four possible states (`yes`, `no`, `partial`, `unknown`). The module also introduces the European EPIAP profile (NeTEx Part 6) and its regulatory implications for Portuguese operators.

- **Module 6 — Fare Structures (Introduction):** Provides an introductory overview of the NeTEx fare modeling system (Part 3). It covers the `FareFrame` structure, `TariffZone` definitions, `PreassignedFareProduct` entities for fixed-price tickets, and `SalesOfferPackage` elements for distribution channels. The Porto Andante ticketing system is used as the primary example, illustrating zone-based fare calculation. The module explicitly delineates its introductory scope, noting that advanced fare modeling (monthly passes, concessions, multi-operator interoperability) falls within the full NeTEx Part 3 specification.

- **Module 7 — Portuguese National NeTEx Profile:** Documents the Portuguese National Profile (PT-EPIP), the country-specific subset of NeTEx mandated by the IMT (Instituto da Mobilidade e dos Transportes) for data submission to the National Access Point (NAP Portugal). The module catalogs mandatory, recommended, and optional fields for each frame type, specifies the IFOPT-based identifier conventions, describes the expected file packaging structure for NAP submission, and provides a comparative analysis with the French national profile. This module addresses a gap in existing NeTEx educational resources, as no equivalent Portuguese-specific documentation was previously available.

- **Module 8 — Practical Tooling:** Equips readers with the practical tools and workflows necessary for production deployment. It covers NeTEx validation using the Entur `netex-validator`, XML editing environments (VS Code with the Red Hat XML extension, Oxygen XML Editor), GTFS-to-NeTEx conversion tools and their known limitations, XSD schema navigation techniques, and a comprehensive pre-submission checklist for the NAP Portugal. The module bridges the gap between understanding the standard and successfully implementing it in an operational context.

Each module includes worked XML examples with extensive inline commentary, practical exercises with annotated solutions, and a formal references section citing all standards, regulations, and data sources used. By combining rigorous standards-based content with an accessible, GTFS-comparative pedagogy and real Portuguese operational data, the documentation enables transport operators and technical practitioners to confidently navigate the transition from GTFS to NeTEx in compliance with the Portuguese National Profile and European interoperability requirements.
